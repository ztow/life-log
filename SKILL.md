# 生命日志 Life Log

## 触发条件

用户说"整理今天的日志"、"整理昨天的日志"、"生成生命日志"、或每天早上8点自动触发。

## 核心流程

### 1. 确定日期
- **手动触发**：用户明确指定日期时用指定日期；否则默认昨天
- **定时触发**：每天早上8点处理"昨天"的文件（即 cron 触发当天往前推一天）
- 特殊场景：用户可以说"整理7月15日的日志"来明确指定
- 输出文件命名格式：`YYYY-MM-DD 生命日志.md`
- **关键**：日志日期≠文件创建日期。日志日期是处理日期（默认昨天），文件时间仅用于时间线排序

### 2. 读取待整理目录
扫描目录：`~/Documents/生活日志/待整理/`

获取目录下所有文件，按**创建时间**排序。

### 3. 逐文件处理

#### 3.1 视频文件（mp4/mov/avi/mkv）
- 用 ffmpeg 每秒提取一帧：`ffmpeg -i input.mp4 -vf fps=1 frame_%03d.jpg`
- 对关键帧（首中尾+间隔抽样）调用 vision_analyze，描述内容
- 视频内容合并为一段文字描述

#### 3.2 录音文件（m4a/mp3/wav）
- 调用 whisper 或系统 TTS API 转写为文字
- 保留完整转写文本

#### 3.3 文本文件（txt/md/json）
- 直接读取内容
- 保留原始文本或关键摘要

#### 3.4 照片（jpg/png/heic/webp）
- 调用 vision_analyze 描述图片内容
- 如果图片文件名或EXIF信息含时间，标注时间

#### 3.5 PDF 文件
- 如果是书籍/文档：提取标题，记录"阅读了《书名》"
- 如果是短内容：提取关键文字摘要

#### 3.6 邮件文字（用户复制的粘贴内容）
- 当纯文本处理，提取关键信息

#### 3.7 含 OCR 的图片
- 当文字处理，提取文字内容

### 4. 时间获取规则

按以下优先级确定每条记录的时间：
1. 文件EXIF/元数据中的创建时间
2. 文件名中的时间戳
3. 文件系统创建时间（stat ctime）
4. 询问用户

### 5. 生成汇总日志

输出路径：`~/Documents/生活日志/汇总/YYYY-MM-DD 生命日志.md`

格式要求：
- 每个条目独立列出，**严格按时间线排序**
- **不合并**任何条目，同一时间有多条则分别列出
- 条目格式：
  ```
  ## HH:MM [类型]
  内容描述...
  ```
- 内容描述要具体：
  - ✓ "讨论了买索尼A7M4，理由是拍摄视频需求，最终决定等双十一降价再买"
  - ✗ "讨论了买哪个设备"
- 书籍只记录：`阅读了《书名》`
- 处理完成后显示汇总统计（多少个文件，多少类型）

### 6. 自动归档

处理完成后，**自动**将待整理目录中的原始文件移到归档目录，**不询问用户**。

归档路径格式（按**日志日期**组织，而不是文件创建时间）：
```
~/Documents/生活日志/归档/
├── 2026/
│   ├── 07/
│   │   ├── 07/
│   │   │   ├── IMG_1234.jpg
│   │   │   └── 录音.m4a
│   │   └── ...
│   └── ...
└── ...
```

按日志日期的**年/月/日**三级目录结构存放，保持原文件名。如果文件已存在于归档目录（同名），在文件名后加时间戳避免覆盖。

### 关键原则

- **日志日期 = 处理日期**（手动默认昨天，定时往前推一天）
- **条目时间 = 文件创建时间**（用于时间线排序和HH:MM标注）
- **归档目录 = 按日志日期**（所有同一日志日期下的文件归档到同一个日期目录）
- **不合并条目**：即使同一个时间点，不同文件分别列出
- **具体化描述**：拒绝模糊概括，要求具体内容

## 分析深度原则

- 具体事件：谁、什么事、过程、理由、结果都要说清楚
- 日常琐事：轻描淡写，一句话带过
- 书籍/长文：只记书名/标题，不抄内容
- 不要过度概括，每个条目要有信息量

## 文件结构

```
~/Documents/生活日志/
├── 待整理/          # 用户每日汇总文件的地方
│   ├── IMG_1234.jpg
│   ├── 录音.m4a
│   └── notes.txt
├── 汇总/            # 生成的日志MD文件
│   ├── 2026-07-07 生命日志.md
│   └── ...
├── 归档/            # 已处理的原始文件（按日志日期归档）
│   └── 2026/
│       ├── 07/
│       │   ├── 06/
│       │   │   ├── 盲王与AGI底层架构.mp4
│       │   │   └── 智能纪要...pdf
│       │   └── 07/
```

## 定时任务

每日早上8点自动执行（cron job），整理前一天的文件。
